## 오늘 한 일

1. 프로젝트 관리 특강 수강
2. 데일리 스크럼
3. Git, JIRA 사용법 코치님 특강
4. 와이어 프레임 회의 (피그마 작성 및 디테일 확인)
5. 회의 때마다 여러 개의 녹음 프로그램으로 녹음해보면서 정확도 확인.(녹음기 어플, 노트북 LM, 에이닷) 어떤 프로그램을 쓰는 게 우리 프로젝트에 가장 적합할 지 좀 더 고민해봐야될 거 같다. 

### 문제가 있었던 것

- 각자가 생각한 디테일이 달라서 아이디어를 모으는데 꽤 시간이 걸렸다.
- 초기에는 어제 만든 각자의 와이어프레임을 가지고 합치려고 했는데 시간이 너무 오래 걸려서, 해빈이가 피그마로 기본 틀을 짜고 거기에 살을 붙이고 빼는 식으로 바꿨다.
- 3D로 구현할지 2.5D로 구현할지 아직 결정하지 못했다. 어떤 방식이 시간적으로 더 효율적일지, 우리 프로그램을 더 잘 나타낼 수 있을지 고민해 봐야겠다.

### 새로 배운 것

- Git의 효율적인 사용법
- 프로젝트 관리법

### 아직 잘 모르는 것, 부족한 것

- 피그마 작성에 익숙했으면 해빈이 도와서 더 수월하게 했을 것 같다. 배울게 너무 많다...

- 3D와 2.5D의 차이를 확실히 알아야겠다.
- 회의할 때 효율적으로 의견 피력을 못한 것 같다. 쿠션어 사용이나, 샌드위치 화법 사용, 내 의견 피력하기 등을 더 연습해야겠다.
